# Expected real files for next phases

## Goal

The next development phases require real exported files from DJI Agras operations (`.dat`, `.txt`, `.kml`, `.csv`) for local validation.

Real files should be used locally only.

## Privacy and repository rules

- Do not commit raw real files.
- Do not commit real coordinates.
- Do not commit identifiable KML routes.
- Do not commit raw DAT/TXT/CSV/KML exports.
- Public examples should be synthetic or sanitized.
- Shifted coordinates are not necessarily anonymous because route shape can still identify a property.
- The safest public sample is synthetic data.

## Sanitization for public examples

```bash
PYTHONPATH=src python -m drone_audit.tools.anonymize_csv \
  --input real_export.csv \
  --output sanitized_export.csv
```

```bash
PYTHONPATH=src python -m drone_audit.tools.anonymize_csv \
  --input real_export.csv \
  --output sanitized_export.csv \
  --fake-coordinates
```

Fake coordinates generated by this command are synthetic and not real.
